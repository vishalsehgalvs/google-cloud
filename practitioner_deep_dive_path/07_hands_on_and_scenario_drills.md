# Hands-On And Scenario Drills

Use this workbook after finishing domain deep-dive files.

## Part 1: Service selection drills

For each prompt, write:

- primary service
- why not the nearest alternative
- one security and one cost control addition

Prompts:

1. Build a stateless API with bursty traffic and low ops budget.
2. Store media files for web app with high durability.
3. Relational app database with managed maintenance.
4. Global analytics over very large data with SQL.
5. Event-driven processing on file upload.

## Part 2: IAM and governance drills

For each prompt, define:

- principal type
- role level
- scope

Prompts:

1. Analytics team needs read-only dataset access.
2. CI pipeline needs to deploy application artifacts.
3. Service needs to publish messages to messaging system.
4. New department needs isolated billing visibility.

## Part 3: Cost and reliability drills

For each prompt, list controls:

- budget
- alert thresholds
- quota
- monitoring and logging actions

Prompts:

1. Product owner wants no month-end billing surprises.
2. SRE team wants faster incident detection.
3. CTO wants proof of reliability posture improvements.

## Part 5: Mock elimination exercise

For every question you practice:

1. Eliminate options with unnecessary manual management.
2. Eliminate options with over-privileged access.
3. Eliminate options mismatched to workload category.
4. Choose managed, secure, and scalable option.

## Self-scoring guide

- 90%+: exam-ready
- 75-89%: revise weak domains and repeat drills
- below 75%: re-read deep-dive domain files before attempting another full drill
